MULTI PC SETUP

他PCでも同じ作業環境を使う

GitHubに手順とプロンプト、Google Drive/Sheetsに実データ、Notionに運用知識を置きます。 他PCではCodexに依頼して、clone、確認、ログイン案内、同期まで進めてもらいます。

GitHub 手順・コード・プロンプト
Google Drive 実データ・素材・台帳
Notion 判断ログ・運用wiki
Local PC ログイン・秘密情報・作業実行

Codexに依頼する文面

初回セットアップ

このPCでeBay輸出のback-office作業環境を使えるようにセットアップして。

GitHubのprivate repoから作業環境を取得し、back-officeのREADME、setup/new-pc-checklist.md、setup/sync-runbook.mdを読んで、必要な初期化を進めてください。

やってほしいこと:
- リポジトリを取得済みか確認。未取得なら取得方法を確認して進める
- back-office/setup/bootstrap-local.sh を実行できる場合は実行する
- .env があるか確認し、なければ .env.example から作る
- .env、tmp、大容量ファイル、個人情報がGitHub管理対象になっていないか確認する
- Chrome、Google Drive、GitHub、Notion、Spreadsheetsの接続状態を確認する
- eBay、Sellsta、Ninja Express、Google Sheets、Notionでログインが必要なものだけ私に教える

秘密情報、Cookie、APIキー、パスワード、注文の個人情報はGitHub、Google Drive、Notionに保存しないでください。

作業開始前チェック

このPCで今日の作業を始める前に、back-office環境を最新化して確認して。

やってほしいこと:
- GitHubの最新状態を取り込む
- 作業対象のREADMEや関連するsetup/ops/promptsを読む
- Chromeで必要なログイン状態を確認する
- Google Drive/Sheets/Notion/GitHubプラグインが使えるか確認する
- 今日の作業に必要なシート、Driveフォルダ、Notionページ、eBay/Sellstaページを開けるか確認する
- 問題があれば、私がするべきログインや認証だけ短く教える

コマンドが必要な場合はCodex側で実行してください。私は文章で依頼するだけにしたいです。

作業終了後の同期

今日このPCで行ったback-office関連の変更を整理して、他PCでも使えるようにGitHubへ反映して。

やってほしいこと:
- 変更差分を確認する
- .env、tmp、大容量ファイル、個人情報、注文の住所、APIキー、Cookieが混ざっていないか確認する
- 手順書、プロンプト、設定テンプレート、コードだけを適切にステージする
- 変更内容が分かるコミットメッセージでコミットする
- GitHubへpushする
- Google Drive/Sheets/Notionに残すべき実データや判断ログがあれば、保存先を整理して教える

push前に、何をGitHubへ反映するか短く確認してください。

環境診断

このPCのback-office環境が、メインPCと同じように作業できる状態か診断して。

確認してほしいこと:
- GitHub repoのremote、branch、最新状態
- back-office/setup/new-pc-checklist.md と sync-runbook.md の内容
- .env が存在し、GitHub管理対象外になっているか
- ChromeでeBay、Sellsta、Ninja Express、Google Sheets、Notionにアクセスできるか
- Google Drive、GitHub、Notion、Spreadsheetsプラグインが使えるか
- Google Drive/Sheets/Notionに実データ、GitHubに手順・プロンプトという分担になっているか

不足があれば「私がやること」と「Codexが続けてできること」に分けて教えて。
