Section 03

収益判定モデル

価格差だけでなく、eBay手数料・関税・Ninja Express物流費を分けて見る。

eBay手数料
  • · `手数料対象額(USD)` = eBay総額 + 売上税見込
  • · `落札手数料(USD)` = FVF率 + 固定手数料
  • · `国際手数料(USD)` = 日本登録セラー標準1.35%
  • · `為替手数料(USD)` と `広告費(USD)` は別列で編集
関税・通関
  • · `HTSコード`、`原産国`、`申告額(USD)` を必須管理
  • · 関税率は商品名から決め打ちしない
  • · `関税見込(JPY)` = 申告額 x 関税率 x FX
  • · `通関/ブローカー(JPY)` は請求実績で上書き
物流費
  • · `Ninja国際按分(JPY)` は実重量/容積重量で按分
  • · `US国内送料(USD)` はeBay Shipping Calculatorか実ラベル
  • · `倉庫手数料(JPY)` と `その他費用(JPY)` を分ける
  • · 見積は実請求が出たら必ず差し替える
最終判定
  • · `純利益見込(JPY)` を最優先に見る
  • · `利益率` は売上総額JPYに対する割合
  • · `必要確認` にHTS未確認、US送料未確認などを残す
  • · 目標利益3,000円 / 目標利益率15%を初期基準にする
Formula
純利益見込(JPY)
= eBay総額(USD) x FX
- 国内価格目安(JPY)
- (落札手数料 + 国際手数料 + 為替手数料 + 広告費)(USD) x FX
- 関税見込(JPY)
- 物流費合計(JPY)
Section 04

Ninja Express連携

コード番号を軸に、国際輸送・在庫・出荷・請求をつなぐ。

商品カタログ
Ninjaコード番号を作り、eBay/SellstaのSKUにも入れる。
Export Form
HTS、原産国、数量、申告額、TTSレートを確認する。
オレゴン入荷
到着日、到着数、写真URL、コメントを在庫へ反映する。
eBay出荷
⑧eBay出荷で出荷指示数、注文番号、追跡番号、US送料を記録する。
返品/US入荷
⑤で事前登録、⑥で荷受反映。不明荷受を減らす。
請求突合
国際送料、燃油、保険、手数料を商品コード別に按分して実績更新。
Section 05

判定ルール

セラーを4段階に分ける。色で覚える。

最
最優先 → 商品取り込みへ
同ジャンルの商品が多い、売価帯が合う、送料無料や日本発送が目立つ。商品取り込みまで進めたいセラー。
候
候補 → もう少し見る
条件は悪くないが、Switch比率や状態に確認余地あり。ランク確認後に判定。
保
保留 → 後回し
気になるが弱い。雑多寄り・Switch少なめ。別ジャンルで再評価することも。
除
除外 → 今回は使わない
評価不足 / ジャンク多い / 雑多すぎ / 自社・大型小売 / 送料高い / Switch関連少ない。
注意するセラー
  • · Rank A/B/C など状態ランク型が多い
  • · Console Only / 付属品違いが多い
  • · or Best Offer が多い
  • · Free returns が多く返品リスクあり
除外寄りのセラー
  • · 評価数50未満
  • · ジャンク / 部品が多い
  • · 商品ジャンルが雑多すぎる
  • · 自社 / 公式 / 大型小売っぽい
  • · 送料が高い商品ばかり
Section 06

安全運用

「やる」と「避ける」を分けて覚える。

DO
やる運用
人間がセラーハックスで検索する
CSV / 画面コピーは人間が取得する
Codexは取得済みデータを整理 / 判定
数値だけでなく目視メモを残す
5〜10セラーまとめてCodexに渡す
DON'T
避ける運用
Codexがログイン済み画面を自動巡回
CodexがeBay / Terapeakを大量に開く
CAPTCHA や制限を回避する
1セラーごとにCodexへ渡す
キーワード結果だけでセラーを決める
